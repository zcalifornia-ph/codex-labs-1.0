# Contributing

Thank you for contributing to CODEX LABS 1.0.

This repository supports workshop learning materials on Python data preprocessing, visualization, and exploratory data analysis.

## Before You Start

- Review open issues and pull requests to avoid duplicated work.
- Keep changes scoped and easy to review.
- Follow `CODE_OF_CONDUCT.md` and `SECURITY.md`.

## What You Can Contribute

- notebook improvements (`codex-labs/*.ipynb`)
- data workflow fixes and clarifications
- documentation improvements in root project files
- reproducible examples for workshop exercises

## Development Flow

1. Fork the repository.
2. Create a focused branch from `main`.
3. Make your changes with clear, testable intent.
4. Validate notebook behavior and data outputs.
5. Open a pull request with:
   - summary of the problem
   - summary of the approach
   - validation notes (screenshots, outputs, or steps)

## Branch Naming

Use concise kebab-case names prefixed by purpose:

- `docs/update-workshop-readme`
- `fix/notebook-merge-step`
- `feat/new-eda-example`

## Commit Messages

Use Conventional Commits where possible:

- `feat: add missing merge notebook example`
- `fix: correct csv load path in concatenate notebook`
- `docs: update workshop preparation links`

## Pull Request Checklist

- change is scoped to one logical concern
- documentation updated when needed
- notebook outputs are consistent and reproducible
- no sensitive information or private data included

## Security Issues

Do not report vulnerabilities in public issues.
Use the process in `SECURITY.md`.
